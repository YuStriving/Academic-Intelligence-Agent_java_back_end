3. **JWT认证**: 使用 Bearer Token 认证方式

## 2. 认证模块详细对接

### 2.1 用户注册 (POST /api/v1/auth/register)

#### 后端响应数据结构
```typescript
{
  code: number;
  message: string;
  data: {
    token: {
      accessToken: string;
      refreshToken: string;
      expiresIn: string;  // ISO 8601 格式日期字符串
      refreshExpiresIn: string;
    };
    user: {
      id: string;
      username: string;
      academicId: string;
      email: string;
      nickname: string;
      avatarUrl: string;
      school: string;
      bio: string;
      gender: string;  // "男"/"女"/"未知"
      createdAt: string;
    };
  };
  timestamp: number;
}
```

#### 前端类型定义对比
```typescript
// ✅ 前端已定义类型完全匹配
export interface AuthResponse {
  token: TokenPairResponse;
  user: UserInfo;
}

export interface TokenPairResponse {
  accessToken: string;
  refreshToken: string;
  expiresIn: string;  // 注意：后端返回的是ISO日期字符串
  refreshExpiresIn: string;
}

export interface UserInfo {
  id: string;
  username: string;
  academicId?: string;
  email: string;
  nickname?: string;
  avatarUrl?: string;
  school?: string;
  bio?: string;
  gender?: string;
  createdAt: string;
}
```

#### 使用建议
- 前端需正确处理 `expiresIn` 字段：这是 ISO 8601 格式的日期字符串，不是秒数
- 如果需要计算剩余秒数，可使用 `new Date(expiresIn).getTime() - Date.now()`

3. 前端需要更新本地存储


- 其他设备保持在线

- 用户需要在所有设备重新登录

#### 添加辅助函数（可选）
```typescript
// src/utils/token.ts
export function getTokenExpiresInSeconds(expiresIn: string): number {
  const expiresDate = new Date(expiresIn);
  const now = Date.now();
  return Math.max(0, Math.floor((expiresDate.getTime() - now) / 1000));
}
```
